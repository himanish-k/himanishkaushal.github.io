Hypertext Transfer Protocol (HTTP) Operation Definitions

There refer to operations between a client (running a browser such as Chrome) and a server (running a web server such as Apache, Nginx etc.).

  • GET - most common; basically means 'get a page'.
  • POST - next most common; typically used to create things (update information).
  • PATCH - used to update information too, but apparently browsers are 'incapable' of sending such requests natively.
  • DELETE - used to destroy data, but browsers are 'incapable' of sending such requests natively too.

From a client's point: Adding a new page to the site

    To add a new page:
  1. Add a route for that page in config/routes.rb .
  2. Add the particular action to the controller (in app/controllers) responsible for the page.
  3. Create a .html.erb file in app/views/<model-name> with the required html page.

Ruby on Rails app creation - a quick run through

    Steps:
  1. Create the new app. 
    	$ rails _<rails-version>_ new <app-name>
  2. Update the Gemfile with the needed bundles and corresponding version numbers and categorize the bundles into groups.
  3. Install the bundles. Generally you want save installing the bundles grouped under production for later (The production stage.). 
    	$ bundle install --without production
  4. Initialize, add files to and commit to a local git repository. 
    	$ git init
	$ git add -A
	$ git commit -m "Initialized repository."
  5. Create online repository on personal git account with the app-name.
  6. Push the changes to the online git repository. 
    	$ git remote add origin git@github.com:<username>/<app-name>.git
	$ git push -u origin --all
  7. Create heroku repository and deploy app. 
    	$ heroku create
	$ git push heroku master

Git

Configuration commands

	$ git config --global user.name "USER NAME"
	$ git config --global user.email "USER EMAIL"

SSH generation commands (to add your repo on an online Git service)

1 generates a file that contains an SSH key.

2 outputs the key in the generated file. This key should be associated with your GitHub account by copying the key and adding it to the account.

3 should output a message confirming that your key was successfully added to the account.

	$ ssh-keygen -t rsa -C "USER EMAIL"
	$ cat ~/.ssh/id_rsa.pub
	$ ssh -T git@github.com

BitBucket

BitBucket is another Git repository holder that allows private repos unlike GitHub but charges only if you exceed a certain number of collaborators.

To initialize, add files, check the status of and commit to a repo

	$ git init
	$ git add -A
	$ git status
	$ git commit -m "Message about added/modified files."
	$ git log

1 initializes an empty git repo within the working directory.
2 adds all the files in the directory to the repo (except the ones listed in the .gitignore file).
3 displays all the files currently added to the local repo.
4 'stages' the changes made in the local repo.
5 shows a log of all commits made to the repo.

To create online repo and push changes to it.

	$ git remote add origin git@github.com:<username>/<repository-name>.git
	$ git push -u origin --all

1 adds the online repo to the project.
2 pushes the changes to the online repo that was just added to the project.

Branching

	$ git checkout -b <branch-name>
	$ git branch
	$ git branch -D <branch-name>

1 creates a new branch with the given name and makes that the current branch.
2 lists all branches and displays the current branch with an asterisk.
